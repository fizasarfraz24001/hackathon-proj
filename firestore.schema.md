## Indexes for Efficient Queries

1. **Users Collection**:
   - By email (for login): `users.email`
   - By creation date: `users.created_at`

2. **Assessments Collection**:
   - By user_id (for user's assessments): `assessments.user_id`
   - By user_id and creation date: `assessments.user_id` + `assessments.created_at`

3. **Recommendations Collection**:
   - By user_id: `recommendations.user_id`
   - By assessment_id: `recommendations.assessment_id`

4. **Learning Paths Collection**:
   - By user_id: `learning_paths.user_id`
   - By recommendation_id: `learning_paths.recommendation_id`
   - By status: `learning_paths.status`

5. **User Progress Collection**:
   - By user_id: `user_progress.user_id`
   - By learning_path_id: `user_progress.learning_path_id`

## Security Rules Example

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can only access their own data
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }

    // Assessments can only be accessed by the user who created them
    match /assessments/{assessmentId} {
      allow read, write: if request.auth != null &&
        request.auth.uid == resource.data.user_id;
    }

    // Similar rules for recommendations, learning_paths, and user_progress
    match /recommendations/{recommendationId} {
      allow read, write: if request.auth != null &&
        request.auth.uid == resource.data.user_id;
    }

    match /learning_paths/{pathId} {
      allow read, write: if request.auth != null &&
        request.auth.uid == resource.data.user_id;
    }

    match /user_progress/{progressId} {
      allow read, write: if request.auth != null &&
        request.auth.uid == resource.data.user_id;
    }
  }
}
```
